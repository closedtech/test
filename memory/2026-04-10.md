## Структура данных BigFloat v2

```python
MAX_DIGITS = 10000  # глобальный лимит

class BigFloat:
    __slots__ = ('digits', 'exp', 'sign', 'max_digits')
    
    # digits: list[int]     # little-endian, BASE=10^9
    # exp: int               # степень старшей цифры (decimal exponent offset)
    # sign: int              # 1 или -1
    # max_digits: int        # ОГРАНИЧЕНИЕ длины — макс. цифр в числе
```

**Формула значения:**
```
value = sign × Σ(digits[i] × 10^(exp - i)) × 10^-(len(digits)-1)
```

**max_digits ограничение:**
- При парсинге: если цифр больше `max_digits` — truncate + round
- При умножении: результат обрезается до `max_digits`
- При делении: round до `max_digits`
- При сложении/вычитании: результат ограничен `max_digits`
- При `sqrt`: Newton итерации до `max_digits` точности

**Пример:**
```python
bf = BigFloat("1" + "0"*20000, max_digits=10000)
# Хранится только 10000 цифр, остальные отбрасываются
len(bf.digits) * 9 <= bf.max_digits  # всегда true
```

## max_digits handling

### Глобальная настройка:
```python
MAX_DIGITS = 10000  # глобальный лимит по умолчанию

class BigFloat:
    def __init__(self, value, max_digits=MAX_DIGITS):
        self.max_digits = max_digits
        # Все операции обрезают до max_digits цифр
```

### Round modes:
- `ROUND_HALF_UP` (по умолчанию)
- `ROUND_FLOOR`
- `ROUND_CEIL`
- `ROUND_TRUNCATE`

### Приоритет max_digits:
```
1. При парсинге: если цифр больше max_digits — truncate + round
2. При умножении: результат обрезается до max_digits
3. При делении: round до max_digits
4. При sqrt: Newton с max_digits цифр
5. При сложении/вычитании: результат ограничен max_digits
```

## Граничные условия (обязательно протестировать)

### Ноль:
- 0 + x = x
- 0 * x = 0
- x / 0 → ZeroDivisionError
- sqrt(0) = 0
- 0^0 → undefined

### Отрицательные:
- (-x) + (-y) = -(x + y)
- (-x) * (-y) = xy
- sqrt(-x) → ValueError (если не поддерживаем комплексные)
- (-x) / (-y) = x/y

### Переполнение (overflow):
- 10^10000 (слишком много цифр для precision)
- Должно обрезаться до MAX_DIGITS

### Underflow:
- 10^(-10000)
- Должно становиться 0 или subnormal
